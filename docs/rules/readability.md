# 可読性ルール

コードの可読性に関するルール。Clean Code、リーダブルコード（The Art of Readable Code）の原則に基づく。

## 1. 命名

- 変数名・関数名・クラス名は意図を正確に伝える名前にすること
- 不必要な略語を使わないこと
- 一貫した命名規則を守ること
- スコープに対して適切な名前の長さにすること（短いスコープなら短い名前でOK）

## 2. 関数・メソッド設計

- 1つの関数は1つの責務に集中させること
- 関数は短く保つこと（目安：30行以上は要注意）
- 引数の数を抑えること（目安：4つ以上は要注意）
- ネストを浅く保つこと（目安：3段以上は要注意）

## 3. 制御フロー

- 早期リターン（ガード節）を適切に使うこと
- 複雑な条件式は名前付き変数や関数に抽出すること
- 不必要な否定条件を避けること

## 4. コメント

- コードで表現できることをコメントで補わないこと（冗長なコメントを避ける）
- 「なぜ」を説明すべき箇所にはコメントを書くこと
- 意図的な省略や一見バグに見える挙動には `NOTE:` コメントで設計意図を記録すること

## 5. 構造・レイアウト

- 関連するコードはまとめて配置すること
- 空行で論理的なブロックを区切ること
- インデントやフォーマットは一貫させること

## 6. 複雑さ

- 不必要に複雑な処理を避けること
- より簡潔に書ける箇所は簡潔にすること
- マジックナンバーや文字列は定数化すること
