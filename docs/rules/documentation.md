# ドキュメント整合性ルール

コード変更に伴うドキュメントの整合性に関するルール。

## 1. コードとドキュメントの一致

- コードで変更した関数名・ファイル名・パラメータはドキュメントにも正しく反映すること
- ドキュメントに記載された手順や設定が実際のコードと矛盾しないようにすること
- 新規追加したファイルや機能には対応するドキュメントを用意すること

## 2. Markdownリンクの正当性

- `[text](path)` 形式の相対パスリンクは対象ファイルの存在を確認すること
- コードブロック内のリンクは検証対象外とする
- URLやフラグメントのみ（`#...`）のリンクは検証対象外とする

## 3. スコープ

- 変更に関連する範囲のみチェックすること（変更と無関係なドキュメントの問題は対象外）
- 推測で不整合を指摘せず、コードを読んで事実に基づいて判断すること
