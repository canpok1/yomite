# ドキュメント整合性ルール

コード変更に伴うドキュメントの整合性に関するルール。

## 1. コードとドキュメントの一致

- コードで変更された関数名・ファイル名・パラメータがドキュメントに正しく反映されているか
- ドキュメントに記載された手順や設定が実際のコードと矛盾していないか
- 新規追加されたファイルや機能に対応するドキュメントが存在するか

## 2. Markdownリンクの正当性

- `[text](path)` 形式の相対パスリンクの対象ファイルが存在するか
- コードブロック内のリンクは検証対象外
- URLやフラグメントのみ（`#...`）のリンクは検証対象外

## 3. スコープ

- 変更に関連する範囲のみチェックする（変更と無関係なドキュメントの問題は対象外）
- 推測で不整合を指摘しない（コードを読んで事実に基づいて判断する）
