セルフレビューを実施します。

## 手順

### 1. コード品質レビュー

`/simplify` を呼び出して、変更コードの再利用性・品質・効率をレビューする。

### 2. ドキュメント検証

ドキュメント検証サブエージェントを呼び出し、コード変更とドキュメント（README、SKILL.md等）の整合性を検証する。

### 3. Markdownリンク検証

変更された `.md` ファイル内の相対パスリンクを検証する：

1. `git diff --name-only origin/main...HEAD` で変更された `.md` ファイルを抽出
2. コードブロック外の `[text](path)` 形式のリンクを解析
3. 相対パスを解決し、対象ファイルの存在を確認（URLやフラグメントのみのリンクは除外）
4. 壊れたリンクを報告・修正

### 4. コミット

修正を行った場合はコミットする。
