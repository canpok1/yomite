# 設計意図コメントのルール

意図的に「やらない」判断や、一見バグに見えるが正しい挙動については、コード内に `# NOTE:` コメントで理由を残す。

## 対象となるケース

- 意図的に処理を省略している箇所
- 一見バグに見えるが正しい挙動
- 外部ツール（レビューbot等）から指摘されやすい設計判断

## 書き方

```bash
# NOTE: <何を意図的にしている/していないか>
# <その理由>
```

## 例

```bash
# NOTE: assign-to-claudeラベルは意図的に除去しない。
# PRがマージされずに処理完了した場合に再処理を可能にするため。
```
