# 設計意図コメントのルール

意図的に「やらない」判断や、一見バグに見えるが正しい挙動については、コード内に `NOTE:` を含む設計意図コメントで理由を残す。
コメント記法は対象言語の標準に従う（例: `# NOTE: ...` / `// NOTE: ...` / `/* NOTE: ... */`）。

## 対象となるケース

- 意図的に処理を省略している箇所
- 一見バグに見えるが正しい挙動
- 外部ツール（レビューbot等）から指摘されやすい設計判断

## 書き方

```text
<言語標準のコメント記法> NOTE: <何を意図的にしている/していないか>
<必要なら次行で理由を補足>
```

## 例

```bash
# NOTE: assign-to-claudeラベルは意図的に除去しない。
# PRがマージされずに処理完了した場合に再処理を可能にするため。
```
