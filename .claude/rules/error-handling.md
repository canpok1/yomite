# エラーハンドリングのルール

スクリプトやスキルの実装では、フォールバック（代替処理）を多用せず、前提条件が満たされない場合はエラー終了させる。

## 方針

- フォールバック処理を多用しない
- 前提条件が満たされない場合は、明確なエラーメッセージを出してエラー終了する
- 条件分岐の増加よりもシンプルさ・可読性を優先する

## 理由

フォールバックを多用すると細かい条件分岐が増え、コードの可読性が低下する。
処理の前提条件が満たされない場合は、代替手段を模索するよりも早期にエラー終了して問題を明示するほうが、デバッグや運用が容易になる。

## 例

```bash
# 良い例: 前提条件を検証してエラー終了
if ! command -v gh &> /dev/null; then
  echo "ERROR: gh コマンドが見つかりません" >&2
  exit 1
fi

# 悪い例: フォールバックで条件分岐が増える
if command -v gh &> /dev/null; then
  # gh を使った処理
else
  # gh が無い場合の代替処理（MCP経由など）
fi
```
